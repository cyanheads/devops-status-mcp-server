# 0.1.0 — 2026-05-30

Initial release of devops-status-mcp-server.

## Added

- **`status_check`** — fetches current component health from any Atlassian Statuspage-compatible endpoint; accepts a built-in vendor slug or a raw Statuspage URL as escape hatch.
- **`status_get_incidents`** — retrieves active and recent incidents for a vendor, including impact level, status, and chronological updates.
- **`status_watch_stack`** — polls health for a named list of vendors in parallel and persists the stack to tenant-scoped state for repeat use; surfaces per-vendor component-level degradations.
- **`status_check_certs`** — performs a real TLS handshake against one or more domains; reports days to expiry (warning < 30 d, critical < 7 d), SAN coverage, chain depth, TLS version (flags 1.0/1.1), cipher suite, and HSTS.
- **`status_check_dns`** — resolves a domain against multiple DNS resolvers in parallel; detects propagation divergence across resolvers and record types (A, AAAA, CNAME, MX, TXT, NS).
- **`status_suggest_action`** — returns structured incident-response guidance for a given status/impact pair; no external API call.
- **`status_list_vendors`** — lists and filters the 26 built-in public vendor entries with their slugs, categories, and Statuspage URLs.
- **`devops-status://vendors/{name}` resource** — reads a single vendor entry by slug with full metadata.
- **Vendor registry** — 26 built-in public vendors across cloud infrastructure, databases, SaaS communications, auth, observability, and AI (DigitalOcean, Linode, Cloudflare, Akamai, GitHub, npm, Vercel, Netlify, Render, Fly.io, MongoDB Atlas, PlanetScale, Supabase, Neon, Redis Cloud, Slack, Discord, Twilio, SendGrid, Mailgun, Auth0, Clerk, Datadog, Sentry, OpenAI, Anthropic).
- **SSRF guard** — `status_check`, `status_get_incidents`, `status_watch_stack`, `status_check_certs`, and `status_check_dns` all block user-supplied raw URLs and domains that resolve to private (RFC 1918), loopback, link-local (169.254.0.0/16), shared address space (RFC 6598), or cloud-metadata ranges for both IPv4 and IPv6; opt-out via `STATUS_ALLOW_PRIVATE_TARGETS=true` for trusted internal deployments.
