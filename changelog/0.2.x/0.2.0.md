# 0.2.0 — 2026-05-30

Breaking rename: `status_*` → `devops_*` tool prefix and `STATUS_*` → `DEVOPS_STATUS_*` env prefix. ([#1](https://github.com/cyanheads/devops-status-mcp-server/issues/1))

## Removed

- **`status_*` tool names** — all 7 tools renamed; saved workflows and configs referencing the old names will stop resolving. No aliases registered — MCP has no native alias mechanism and duplicating tools would double the surface this change cleans up.

## Changed

- **Tool names** renamed to carry the `devops_` namespace prefix:

  | Old | New |
  |:--|:--|
  | `status_check` | `devops_status_check` |
  | `status_check_certs` | `devops_check_certs` |
  | `status_check_dns` | `devops_check_dns` |
  | `status_get_incidents` | `devops_get_incidents` |
  | `status_list_vendors` | `devops_list_vendors` |
  | `status_suggest_action` | `devops_suggest_action` |
  | `status_watch_stack` | `devops_watch_stack` |

- **Env var prefix** renamed from `STATUS_*` → `DEVOPS_STATUS_*`:

  | Old | New |
  |:--|:--|
  | `STATUS_CACHE_TTL_MS` | `DEVOPS_STATUS_CACHE_TTL_MS` |
  | `STATUS_FETCH_TIMEOUT_MS` | `DEVOPS_STATUS_FETCH_TIMEOUT_MS` |
  | `STATUS_CERT_TIMEOUT_MS` | `DEVOPS_STATUS_CERT_TIMEOUT_MS` |
  | `STATUS_DNS_TIMEOUT_MS` | `DEVOPS_STATUS_DNS_TIMEOUT_MS` |
  | `STATUS_ALLOW_PRIVATE_TARGETS` | `DEVOPS_STATUS_ALLOW_PRIVATE_TARGETS` |

  `src/utils/ssrf-guard.ts`, `.env.example`, `server.json`, and `manifest.json` updated alongside the config schema. Test files renamed and updated to match.
