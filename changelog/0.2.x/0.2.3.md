# 0.2.3 — 2026-06-12

## Added

- **`devops_get_incidents` truncation enrichment** — emits `truncated`, `shown`, and `cap` via `ctx.enrich.truncated()` when more incidents match than the `limit` returns, with guidance to raise the limit (max 50) or filter by status.
- **Display identity** — `createApp()` now sets `name` and `title` explicitly to `devops-status-mcp-server` so clients show the machine name rather than the npm scope.
- **MCPB bundle cleaner** (`scripts/clean-mcpb.ts`) — post-pack step wired into `bundle`: runs `mcpb clean` (dev-dependency prune) then strips dependency-shipped agent-doc entries (`node_modules/**` `skills/`, `.claude/`, `.agents/`, `SKILL.md`) that root-anchored `.mcpbignore` patterns cannot reach.
- **`coerce-boolean-env-flag` antipattern rule** — `check-framework-antipatterns.ts` flags `z.coerce.boolean()` on env flags (`Boolean("false")` is `true`); comment lines are skipped so rule docs don't self-trip.
- **Packaging lint coverage** — `lint-packaging.ts` adds `.mcpbignore` dev-dir/runtime-path probes, post-bundle content assertions (shared `AGENT_DOC_ENTRY` with the cleaner), and `createApp()` `name`/`title` identity checks.

## Changed

- **Error-contract codes** recoded from `InvalidParams` to semantic codes across the tool surface: `vendor_not_found` (`devops_status_check`, `devops_get_incidents`, `devops_watch_stack`) and `no_stack` (`devops_watch_stack`) → `NotFound`; `invalid_domain` (`devops_check_certs`, `devops_check_dns`) and `target_blocked` (all SSRF-guarded tools) → `ValidationError`.
- **`.mcpbignore`** patterns root-anchored (`skills/` → `/skills/`, etc.) so they match only top-level entries, not nested `node_modules/**` paths.

### Dependencies
