# 0.2.5 — 2026-06-20

## Changed

- **`@cyanheads/mcp-ts-core` `^0.10.6` → `^0.10.9`** — picks up:
  - **`ctx.content`** collector on `Context` for non-text content blocks — image/audio bytes prepend to `content[]` after `format()` and never enter `structuredContent` ([mcp-ts-core #239](https://github.com/cyanheads/mcp-ts-core/issues/239)).
  - **Canvas SQL gate** now classifies a `SELECT`-shaped statement that prepares-fails (mistyped column, unknown function) as `ValidationError` with `data.reason: 'invalid_sql'` and the DuckDB binder detail in `data.binderMessage`, instead of the misleading `non_select_statement` ([mcp-ts-core #236](https://github.com/cyanheads/mcp-ts-core/issues/236)).
  - **`DuckdbProvider.describe({ tableName })`** qualifies its pushed `WHERE` filters with the `t` alias, fixing a DuckDB ambiguous-column Binder Error on every filtered `describe()` ([mcp-ts-core #235](https://github.com/cyanheads/mcp-ts-core/issues/235)).
  - **`OTEL_SERVICE_NAME`** docs corrected: `createApp({ name })` only seeds it when unset (env → `createApp` `name` → `package.json` `name`) ([mcp-ts-core #233](https://github.com/cyanheads/mcp-ts-core/issues/233)).
- **`devcheck.config.json`** — added `packaging.pluginManifests: true`, enabling the framework's new `lint:packaging` check 10 (validates `.claude-plugin/plugin.json`, `.codex-plugin/plugin.json`, `.codex-plugin/mcp.json` descriptions + scoped/unscoped identity) ([mcp-ts-core #240](https://github.com/cyanheads/mcp-ts-core/issues/240)).
- **`scripts/check-dependency-specifiers.ts`** — new framework devcheck step (`Dependency Specifiers`) that hard-fails on floating specifiers (`latest`, `*`, pre-release dist-tags) in `package.json` and the `bun.lock` workspaces map ([mcp-ts-core #246](https://github.com/cyanheads/mcp-ts-core/issues/246)).
- **Synced 14 `skills/*` and 6 `scripts/*`** to their `^0.10.9` framework versions (`git-wrapup` v1.4, `orchestrations` v1.4, seven `api-*`/`field-test`/`report-issue-local`/`tool-defs-analysis` body re-syncs, fresh-scaffold devcheck guards).

### Dependencies

- `@cyanheads/mcp-ts-core` `^0.10.6` → `^0.10.9`
- `typescript` `^5.9.3` → `^6.0.3`
- `@types/node` `^25.9.3` → `^26.0.0`
